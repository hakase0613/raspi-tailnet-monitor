# UI 状态色态语义 (iter2)

监控页面对每个指标卡片使用四种基础色态：`ok` / `warn` / `bad` / `neutral`，并在
此基础上**用非颜色特征**区分两类容易被误读的特殊状态：**真实空闲**（`idle`）
与**无数据**（`na`）。这份文档说明三者的含义与视觉规则，供色弱 / 高对比度 /
打印场景下的阅读对照。

## 色态总览

| 色态      | 含义                       | 文本提示       | 边框       | 颜色基底                |
|-----------|----------------------------|----------------|------------|-------------------------|
| `ok`      | 数据正常、值在阈值以下     | 正常 / 直连等 | 实线，绿色 | `--ok` 绿               |
| `warn`    | 阈值附近，需关注           | 偏高 / 关注    | 实线，黄色 | `--warn` 琥珀           |
| `bad`     | 越界或离线                 | 过热 / 离线    | 实线，红色 | `--bad` 红              |
| `neutral` | 没采到 / 不适用 / 未配置   | -- / 无数据    | 实线，灰色 | `--muted` 灰            |
| `idle`    | 实测为 0（空闲），数据有效 | 空闲 / 0%      | **虚线**   | `--accent` 蓝（淡）     |
| `na`      | 缺失值（采集失败/不可用）  | -- / 失败      | 实线，灰   | `--line-strong`         |

## `ok` vs `idle`：为什么要再分一档

很多卡片绿色 (`ok`) 表示"指标在安全区"，但 0% / 0°C 也会落在安全区，仅看颜色
会误以为"组件在工作得很轻松"。`idle` 在保留绿色基调的同时加了：

1. **虚线边框**：`border-style: dashed`，肉眼一眼能从一排实线卡片中挑出。
2. **圆点 icon**（`•`）：在标签前置一个 `--accent` 颜色的圆点，色弱用户也能区分。
3. **文案"空闲"**：在副标题或副位面文本中显式追加"空闲"二字。
4. **较冷色**：从绿色过渡到 `--accent` 蓝（dial / bar fill 都会切换），保留
   "数据正常"含义但和 `ok` 的实工作绿区分开。

DOM 上的对应属性是 `data-idle="1"`：
- `.thermo[data-idle="1"]` → 温度温度计 0°C 的安全空闲读数
- `.bar-row[data-idle="1"]` → 横条 0%
- `.mini-gauge[data-idle="1"]` / `.gauge-card[data-idle="1"]` → 圆环表盘 0%

## `na` vs 其他

`data-na="1"` 表示**真的没拿到数**（hwmon 缺失、SSH 失败、字段为 null）。条/盘
直接显示成实线灰，文案为 `--`。它和 `idle` 的区别：

- `idle`：数据来了，且值合法地等于 0 → 蓝色虚线 + "空闲"
- `na`：数据没来 → 灰色实线 + "--"

## "采集滞后"标签

顶栏的"最后采集 X 秒前"会在 `(now - updated_at) > poll_interval × 2` 时把
pill 变灰并附加红黄色的"采集滞后"小徽标。徽标 CSS 由 `.update-pill.lagging`
驱动，配合 `metaText` 文本里的 ` · 采集滞后`。这条规则用于识别
后端进程僵死、cron 漂移、SSH 全表超时这类**没有显式错误但已经停摆**的情况。

## GPU "计算空闲"

NVIDIA / Tesla 卡片有个微妙状态：`utilization_gpu_percent === 0` 但
`memory_used_mib > 0`，表示模型还驻留在显存里、但当下没有活跃推理。
此时文案改写为 `显存占用 N MiB（计算空闲）`，并在 chip 区追加
`计算空闲` 标签，卡片整体边框改为虚线（`.gpu-card[data-compute-idle="1"]`），
避免把它和"GPU 已掉线 / 模型已卸载"混淆。

## 维护建议

- 新增传感器 / 新增指标时，请优先按上面的色态语义复用现有 `data-*` 属性，不要
  发明新的色 token（保持 a11y 一致性）。
- 改 `static/app.js` 中的 `humanTempLabel` 或 `tempBucket` 后，请同步更新
  `scripts/test-temp-labels.mjs` 的用例，保证 `node scripts/test-temp-labels.mjs`
  仍然全绿。
- 需要改 idle / na 的视觉时，集中改 `static/style.css` 末段的 iter2 块，避免
  和老规则颜色冲突。
